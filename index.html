Horrorscope

Horrorscope is a fun app that automatically generates horoscopes, with a twist. Following in the style of the early '00s horoscope sites, Horroscope looks as it is built, tongue in cheek. Horrorscope was built as a solo project in just over four days during the General Assembly Web Development Immersive.


Horrorscope is built primarily in Ruby on Rails (standard MVC fun). Horrorscope relies on javascript for corny animations and AJAX calls to external APIs for the movie content displayed. The front end was written in HTML5 and CSS3, using Sass and Bourbon.

WDIConf

WDIConf was built (in around 5 days) for the General Assembly WDI annual conference. WDIConf is built using Ruby on Rails with Sinatra mounted to serve APIs from the back end to a Backbone.js SPA on the front end and Rails running admin/backstage CRUD.


A group project with 7 talented team members, WDIConf was an excellent learning experience. My role in the team was Project Manager, though I contributed throughout the codebase.

Huggers

Huggers is a dating app for people who want relationships and not hookups, or for people already in a relationship. It's also a project completed midway through the Web Development Immersive and the first project I wrote in Ruby.


Huggers is written in Ruby and uses Sinatra to serve data to the front end including a funky AJAX/jQuery combo to build the list view served on a search request.


Written mobile first, Huggers is optimally viewed on a small screen.

PastaMasta

PastaMasta exists to help everyone who looks at a Jamie Oliver 30 minute meal and breaks into a cold sweat. PastaMasta provides simple pasta recipes in a mobile first responsive web app for the non-cook.


PastaMasta was a WDI group project that provided a steep learning curve in version control and effective seperation of duties in group programming. Built in Ruby on Rails, PastaMasta uses the Weeatt API for recipes and serves recipes via internal API to front end AJAX calls for a smooth experience.
